# 建筑板部工具箱

用于处理「建筑板」与「冷库板」概览表及相关 Excel 的内部桌面工具（Windows / macOS）。基于 Tauri + React，业务计算由本地 Python sidecar 完成，数据全程在本机处理。

## 功能

左侧分 **建筑板 / 冷库板** 两条业务线，各有「概览表生成」「概览表更新」两张卡片。

- **概览表生成（建筑板）**：合并多份装箱明细 → 核对成品物流并标黄缺货 → 写入生产状态 → 追加下单数量差额，一步导出带「汇总」「每日发货数据」的概览表。
- **概览表生成（冷库板）**：冷库板系统里没有装箱明细，改以成品物流逐行展开为已生产板材行（真实唛头 / 出库日期 / 数量），再用下单详情补未生产余量占位；「板材明细」带「外板颜色」「内板颜色」两列。
- **概览表更新**：在已有概览表上增量更新，按需补传成品物流 / 切锯表 / 下单详情 / 装箱明细，传哪个刷新哪一块；都不传则只重算汇总与每日发货。
- **从系统拉数**：系统数据接口免登录，依据下单详情（或概览表）中的库号与项目名，一键拉取装箱明细（仅建筑板）、切锯表、成品物流并自动填入对应文件槽。「高级」区可调车间、切锯范围与物流时间窗——成品物流单次最多查近 180 天，跨度更长的项目可再填一个更早的结束日期，自动分两段各查近 180 天、合并去重。
- **汇总 / 每日发货**：各状态面积按 板宽 + 颜色 + 板厚（冷库板按 外板颜色 + 内板颜色）统计；并按出库日期统计每日发货面积。
- 概览表「板材明细」含「合同编号」列（取自下单详情，固定置于第 2 列）。

## 运行环境

- 检测到 Microsoft Excel 时走 COM 原生重算（excel-native），面积 / 汇总公式缓存即时回写；否则回退 openpyxl-only（用 Excel 打开会自动重算）。
- 下单详情支持 xlsx / xlsm，以及 UTF-8 / GBK 编码的 CSV。

## 安装与更新

- Windows 提供签名安装包与便携版，macOS 提供 dmg，见 [Releases](https://github.com/lyq2010/building-panel-dept-toolkit-releases/releases/latest)。
- 应用内自动更新：启动时检查新版本，装机版走安装包更新，便携版原地替换 zip。
